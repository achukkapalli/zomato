# Streamlit Community Cloud Deployment Plan

This guide outlines the steps to deploy the **Zomoto AI Recommendation Monolith** to [Streamlit Community Cloud](https://streamlit.io/cloud) (Streamlit Sharing).

---

## 📋 Prerequisites

1. A **GitHub account** with the project code pushed to a repository.
2. A **Streamlit Community Cloud account** linked to your GitHub account.
3. A **Groq API Key** (from [Groq Console](https://console.groq.com/keys)).

---

## ⚡ Step 1: Optimize Data Cache for Streamlit Sharing (Recommended)

Streamlit Community Cloud instances have CPU and memory constraints (~1 GB RAM). Pre-processing the 51,717 restaurant records on a fresh server boot can cause slow startup times or Out-Of-Memory (OOM) crashes.

To ensure **instant load times (< 100ms)** and bypass Hugging Face downloads during startup, commit your pre-processed local cache snapshot to Git.

Since `.gitignore` excludes `.parquet` files and the `data/cache/` directory, force-add the cache file to git:

```bash
# Force add the preprocessed cache file
git add -f data/cache/restaurants.parquet

# Commit the cache snapshot
git commit -m "chore: include preprocessed restaurants cache for deployment"
```

*Note: The cache file is only ~1.8 MB, making it extremely lightweight for Git.*

---

## 📤 Step 2: Push Code to GitHub

Ensure all your latest local changes are committed and pushed to your remote repository branch:

```bash
git add .
git commit -m "feat: complete phase 6 testing and ui polish"
git push origin main
```

---

## 🚀 Step 3: Deploy on Streamlit Community Cloud

1. Navigate to [share.streamlit.io](https://share.streamlit.io/) and log in with your GitHub account.
2. Click the **"New app"** button in the top right.
3. Configure the deployment settings:
   * **Repository**: Select your `zomoto` repository.
   * **Branch**: Select `main`.
   * **Main file path**: Specify `src/app/main.py`.
   * **App URL**: (Optional) Customize the subdomain prefix if desired.

---

## 🔑 Step 4: Configure Groq API Key (Secrets)

Before clicking deploy, you must configure your application secrets so the AI recommender can talk to Groq.

1. Click on **"Advanced settings..."** at the bottom of the deploy form.
2. In the **Secrets** text box, enter your secrets in TOML format:

```toml
# Inject Groq API credentials
GROQ_API_KEY = "gsk_your_actual_groq_api_key_here"

# Optional environment overrides
LLM_MODEL = "llama-3.3-70b-versatile"
MAX_CANDIDATES = 30
TOP_K = 5
DATA_SNAPSHOT_PATH = "data/cache/restaurants.parquet"
```

3. Click **"Save"**.
4. Click **"Deploy!"**.

---

## 🔍 Step 5: Verification & Smoke Testing

Once deployment completes, verify the following:

1. **Successful Initialization**:
   * The app should load without errors and show the custom utensils icon next to "Zomoto".
2. **Interactive Elements**:
   * The sliders, textareas, and dropdown menus should render correctly.
3. **Chip Suggestions Test**:
   * Click **"Italian in Indiranagar"**. The system should immediately populate the inputs, display a spinner, and output 5 ranked recommendations with explanations generated by the Groq LLM.
4. **Fallback Handling**:
   * If you temporarily remove/change the `GROQ_API_KEY` secret, the application should display fallback recommendations ranked by rating with a warning banner, instead of crashing.

---

## 🛠️ Troubleshooting

### 1. App takes several minutes to start or crashes on startup
* **Cause**: The Parquet cache was not committed to Git, causing the app to download the entire Zomato dataset from Hugging Face on start.
* **Fix**: Run `git add -f data/cache/restaurants.parquet` locally, commit, and push to GitHub.

### 2. "AI ranking unavailable" banner appears
* **Cause**: `GROQ_API_KEY` is either missing from Streamlit secrets, or contains a typo/whitespace.
* **Fix**: Go to the app control panel in Streamlit Cloud, open **Settings** -> **Secrets**, check the TOML key spelling and value, and save.
